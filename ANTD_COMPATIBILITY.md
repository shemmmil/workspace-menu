# Совместимость с Ant Design

## Проблема

Виджет не открывался в проектах, где используется Ant Design. Проблема возникала из-за:
1. Конфликта z-index значений между компонентами виджета и Ant Design
2. Глобальных стилей Ant Design, влияющих на компоненты виджета
3. Перехвата событий клика компонентами Ant Design

## Решение

### Полная изоляция виджета

Реализована комплексная система изоляции:

#### 1. **Изолированный Portal контейнер**
   - Создан специальный DOM-контейнер `#workspace-menu-portal-root`
   - Все порталы (Dialog, DropdownMenu) рендерятся в изолированный контейнер
   - Контейнер автоматически создаётся при монтировании виджета
   - Расположен вне DOM-дерева родительского приложения

#### 2. **Увеличенные z-index значения**
   - Portal root container: `z-index: 2000`
   - DialogOverlay: `z-index: 2000`
   - DialogContent: `z-index: 2001`
   - DropdownMenuContent: `z-index: 2000`
   - DropdownMenuSubContent: `z-index: 2001`

#### 3. **CSS изоляция с reset**
   - Использование `all: revert` для сброса глобальных стилей
   - Специальные правила для `[data-workspace-menu-portal-root]`
   - Изоляция через CSS свойство `isolation: isolate`
   - Переопределение box-sizing для всех элементов виджета

#### 4. **Изоляция событий**
   - Добавлен `stopPropagation()` на все обработчики кликов
   - Предотвращение всплытия событий к родительскому приложению
   - Защита от перехвата событий компонентами Ant Design

### Таблица z-index

| Компонент                | z-index | Описание                              |
|--------------------------|---------|---------------------------------------|
| Ant Design Modal         | 1000    | Модальные окна Ant Design            |
| Ant Design Dropdown      | 1050    | Выпадающие меню Ant Design           |
| Ant Design Tooltip       | 1060    | Подсказки Ant Design                 |
| **Widget DialogOverlay** | **2000**| **Overlay для модальных окон виджета**|
| **Widget DialogContent** | **2001**| **Контент модальных окон виджета**   |
| **Widget DropdownMenu**  | **2000**| **Выпадающие меню виджета**          |

## Использование

После обновления пакета виджет будет корректно работать в проектах с Ant Design:

```bash
npm install @corp/workspace-menu@latest
```

или

```bash
npm update @corp/workspace-menu
```

## Проверка

После установки обновлённой версии:

1. Виджет должен открываться поверх всех компонентов Ant Design
2. Модальное окно "Все сервисы" должно отображаться корректно
3. Выпадающее меню должно работать без конфликтов

## Дополнительная настройка (если требуется)

Если в вашем проекте используются кастомные z-index значения выше 2000, вы можете переопределить CSS переменную:

```css
[data-workspace-menu-portal] {
  --workspace-menu-z-index: 3000; /* ваше значение */
}
```

## Техническая информация

### Архитектура изоляции

```
document.body
├── [ваше приложение с Ant Design]
└── #workspace-menu-portal-root (z-index: 2000)
    └── [data-workspace-menu-portal]
        ├── Dialog (z-index: 2001)
        └── DropdownMenu (z-index: 2000)
```

### Используемые технологии

- **Radix UI Portal**: изолированный контейнер для рендеринга
- **Custom Portal Manager**: управление жизненным циклом контейнера
- **CSS Reset**: `all: revert` для сброса глобальных стилей
- **Event Isolation**: `stopPropagation()` для изоляции событий
- **CSS Variables**: гибкая настройка z-index
- **Tailwind CSS**: стилизация компонентов

### Ключевые файлы

- `src/lib/portal-container.ts` - управление изолированным контейнером
- `src/components/ui/dialog.tsx` - изолированный Dialog
- `src/components/ui/dropdown-menu.tsx` - изолированный DropdownMenu
- `src/styles.css` - CSS правила изоляции
